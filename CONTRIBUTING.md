# Contributing

## Repo Role

AI bedtime-story MVP

## Working Boundary

Use this repo for product work, subscription enforcement, prompt quality, and launch readiness. Keep changes tightly scoped and avoid speculative platform work.

## Branching

- Branch from `main` unless the repo documentation says otherwise.
- Use short-lived branches with clear names such as `feat/...`, `fix/...`, `docs/...`, or `chore/...`.
- Do not mix unrelated work in a single branch or PR.

## Tracking Expectations

- Review `CURRENT_ISSUES.md` before starting work.
- Open or update an issue before making non-trivial changes.
- Keep PRs small enough to review quickly.
- Document follow-up work instead of hiding it in chat or commit messages.

## Verification

- Run the relevant verification for the changed area. Baseline command: `npm run lint`.
- Call out what was not verified if local setup or credentials block full testing.

## Documentation

- Update the README, status docs, or operational notes when behavior or project direction changes.
- Prefer explicit notes over tribal knowledge.
